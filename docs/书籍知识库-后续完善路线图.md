1. 对当前 92 个有证据概念逐页复核定义、别名和领域归属；

### P1：APP 检索接入

1. 建立“用户问题 → APP 意图 → 概念 → 一跳关系 → 原典证据”的检索链；

### P2：存储与运维

1. 在文件产物验收后再设计数据库表和迁移；
2. 保留稳定 ID，使 Markdown、`graph.json` 和数据库记录可互相映射；
3. 增加增量构建，只重建发生变化的书、概念和关系；
4. 增加图谱差异报告，显示新增、删除、改名和证据变化；
5. 后续可选择性增加 embedding，但不能替代证据回链和图谱约束。

## 每轮固定验收

- 输入书籍数、页数和章节数一致；
- 所有知识节点至少有一条原典证据；
- 所有关系端点、Markdown 文件和段落锚点存在；
- 相同输入重建后的稳定 ID 与排序不变；
- 每本书抽查开头、中间、结尾段落；
- PDF 抽查原图、OCR 和最终证据页三者一致；
- 运行 Ruff 与完整 API 测试；
- 更新 `knowledge_wiki/quality-report.md` 和本路线图的当前基线。
