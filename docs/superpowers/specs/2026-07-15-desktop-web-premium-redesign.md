# 天机寺桌面 Web 高级视觉重设计规格

日期：2026-07-15

## 1. 重设计目标

推翻当前桌面版以 CSS 几何图形绘制寺院、山峰与签筒的视觉方法，保留已经完成的路由、匿名会话、API 接入和业务功能，重新建立具有真实空间、材质、光影与编辑级排版的消费级产品界面。

目标不是“更古风”，而是达到高级文化品牌、数字文博展览和东方精品酒店线上体验的视觉完成度。界面应安静、克制、有真实质感，不像管理后台、PPT 信息图、古风游戏或模板化落地页。

## 2. 已确认的问题

1. CSS 拼出的寺院与签筒缺少透视、材质与可信光影，观感卡通且廉价。
2. 首页标题占比失控，在部分宽屏比例下出现破坏性的断行。
3. 求签页左侧功能列表具有后台菜单感，英文副标题生硬且不服务内容。
4. 大面积无意义空白没有形成张力，只暴露了资源质量不足。
5. 浏览器默认蓝色焦点框破坏品牌一致性。
6. 当前页面把“古风”理解成宋体、红色和装饰线，缺少真正的空间和材质依据。

## 3. 视觉方向：暗室见光

整个产品以光线组织层级。大面积环境保持深、静、低饱和；用户可以操作或需要关注的地方由烛火、门缝、天光或旧铜反光照亮。

视觉关键词：电影感、东方建筑、黑漆木、旧铜、石阶、薄雾、克制、可呼吸。

明确排除：CSS 几何插画、霓虹赛博、仙侠粒子、满屏宣纸纹理、红金堆叠、玻璃拟态、后台卡片矩阵和夸张书法字体。

## 4. 资源标准

### 4.1 核心场景资源

核心页面使用专门生成并经过裁切、调色的高分辨率场景资源，不再用 CSS 绘制主体：

- 首页：16:9 横向寺院中轴场景，山门内视角，暮色蓝调，主殿暖光。
- 求签：16:9 横向暗色签房，黑漆木案、真实竹签筒、侧向天光与香烟。
- 藏经阁：横向文献展厅或木构书阁，局部阅读灯光。
- 问卦：低干扰书房背景，可使用局部材质图而非完整场景。
- 供灯：近景铜灯与火焰，背景保持深色虚化。

资源不得含有生成式模型常见的乱码匾额、错误建筑结构、多余器官、不可解释物件和伪书法。匾额与界面文字由前端叠加，不让图片模型生成文字。

每张主视觉至少输出 2560px 宽版本，并派生 1920px、1440px 和移动裁切版本。统一使用 WebP/AVIF，设置明确焦点区域和安全裁切区。

### 4.2 材质资源

允许使用低对比度的真实材质：黑漆木、氧化铜、石材、棉纸纤维与薄雾。材质仅用于局部加强空间感，透明度保持克制，不能覆盖正文造成脏污。

### 4.3 字体与图标

- 中文展示：思源宋体或 Noto Serif SC，使用 600–700 字重，控制字距。
- 中文正文与界面：思源黑体或 Noto Sans SC。
- 英文与数字：Cormorant Garamond 仅用于展签式编号；Inter 用于数据和状态。
- 图标采用统一的细线系统，主要使用 Lucide 或自制 1.5px 线性图标。
- 删除无意义的英文技术文案；英文只用于展签、编号和日期辅助信息。

## 5. 色彩系统

- 夜墨 `#101514`：全局深背景与顶部导航。
- 烟青 `#354945`：环境层次、次级背景。
- 黑漆 `#201B18`：木作与暗室容器。
- 漆红 `#5F211C`：极少量品牌动作与印记。
- 旧铜 `#B08A4B`：焦点、选中、路径与金属细节。
- 骨白 `#E6E0D2`：正文、纸张与高对比文字。
- 烟灰 `#8D938D`：说明与非活动状态。

任何页面同时出现的强调色不超过两种。旧铜用于高级感和焦点，漆红只用于高价值确认动作。蓝色不进入品牌界面。

## 6. 首页

### 6.1 构图

首页使用全视口电影级寺院场景。镜头位于山门内，沿中轴望向主殿：近景石阶和松枝，中景点灯殿宇，远景山色与雾。

导航覆盖在画面顶部，使用纯深色或随滚动出现的实色背景，不使用玻璃拟态。

主标题改为“入山 · 问心”，放置在左下安全区域，桌面字号限制在 64–76px，最多两行，任何宽度不允许单字孤行。副文案控制在两句内。

今日寺语作为展览图注置于右下或画面边缘，字号小、对比克制，不与主标题争夺注意力。

### 6.2 交互

建筑入口使用细小的旧铜定位线和文字标记。默认只露出殿名；悬停后显示一句动作。热点不使用发光圆点和卡片气泡。

页面仅保留三个动效：极慢的 1–2% 推镜、薄雾水平移动、局部灯火呼吸。减少动态模式下全部静止。

向下滚动进入“今日可做”内容区，使用编辑排版与真实局部资源，不采用四张相同白卡。

## 7. 求签页

### 7.1 取消后台结构

删除左侧殿堂列表、英文模块名、白色大框和 CSS 签筒。页面进入后直接置身暗色签房。

殿堂选择放在画面底部的横向铭牌中：文殊、月老、财神、天机、签房。铭牌只包含一个汉字、殿名和一句主题；选中态用旧铜细边与轻微背景变化，不出现浏览器蓝框。

### 7.2 仪式阶段

签筒位于木案中央，由专门场景资源呈现。选择殿堂后，主文案变化，但场景保持稳定。主要动作只有一个：“静心摇签”。

执行时进行一次 1.2–1.8 秒的镜头推进、竹签轻微位移与木案光线变化。禁止持续抖动整个签筒或使用粒子爆炸。

### 7.3 结果阶段

签文结果从右侧或案面上展开，形成“签纸 + 解读”的非对称布局：

- 左侧窄签纸：签号、签诗、签级。
- 右侧正文：照见、古籍出处、现实提醒、继续问卦。
- 背景仍保留暗室，不切换成另一套白色页面。

签纸使用细棉纸材质与真实阴影。内容排版优先，装饰不超过一个印章和一条竖向题签。

## 8. 其他页面统一规则

- 问卦：保持三栏信息架构，改为安静书房和文献旁证，不做纯色后台侧栏。
- 藏经阁：采用文献展厅视角和大幅书影，书目像展陈目录而非商品卡片。
- 香火殿：真实香案与烟气资源，计时器融入场景，不画 CSS 香。
- 许愿池：使用真实木牌局部与排版信息流，避免九宫格便利贴。
- 命盘：改为深色东方星盘桌，表单独立为骨白编辑页。
- 供灯：使用铜灯主视觉，权益说明克制，不采用电商套餐卡模板。

## 9. UI 细节与可访问性

- 全站 `:focus-visible` 使用 1px 旧铜外框与 3px 暗色间隔，禁止默认蓝框。
- 正文对比满足 WCAG AA，图片上的文字必须有局部暗色承托或安全区域。
- 按钮高度 44–48px，主动作使用漆红实色，次动作使用旧铜描边。
- 弹层使用实色深背景，不用模糊玻璃。
- 加载状态使用门缝光或印章式进度，不使用通用旋转圆环。
- 错误提示直接说明问题和恢复动作，不使用玄学化模糊文案。

## 10. 技术实现边界

保留当前 React 路由、匿名会话、API 客户端、数据类型和业务逻辑。重构范围集中在：

- 首页与核心页面 DOM 结构；
- 视觉资源与响应式裁切；
- 全局设计令牌；
- CSS 组件和动效；
- 求签页面的仪式状态；
- 全站焦点、加载与错误状态。

删除当前不再使用的 CSS 山峰、寺庙、签筒和香火绘图代码，避免旧样式残留。

## 11. 验收标准

1. 首页首屏不再出现任何可辨认的 CSS 几何寺院主体。
2. 1920×1080、1440×900、1280×800 下主标题均不出现单字孤行或遮挡。
3. 求签页不存在左侧后台菜单和默认蓝色焦点框。
4. 首页与求签页的核心场景资源在 Retina 屏幕下保持清晰。
5. 所有核心页面使用同一色彩、字体、按钮、焦点和材质系统。
6. 资源加载前有稳定占位，不产生明显布局跳动。
7. 页面支持键盘操作和减少动态效果设置。
8. TypeScript、生产构建和后端测试全部通过。
9. 浏览器截图复审必须覆盖首页、求签、问卦、藏经阁和 900px 窄屏。

## 12. 实施顺序

1. 生成并筛选首页、签房、藏经阁、供灯核心视觉资源。
2. 重建全局设计令牌、字体和焦点系统。
3. 重做首页与求签页，先完成两张关键截图验收。
4. 统一问卦、藏经阁、香火、许愿、命盘和供灯页面。
5. 输出多规格资源、响应式裁切与性能优化。
6. 完成浏览器视觉复审、构建与测试。
