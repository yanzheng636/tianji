# 古籍知识图谱 V2：完整底本、结构深化与 APP 检索

日期：2026-07-15

## 目标

在 V1 的 13 本书、概念/方法/规则/案例和原典证据层之上，完成四项升级：

1. 复核《麻衣相法》竖排 OCR，并保持页级质量状态；
2. 用可追溯的完整 PDF 底本替换《卜筮正宗》《柳庄相法》的残缺提取来源；
3. 将方法、规则和流派差异结构化；
4. 将 APP 检索改为“意图 → 概念 → 一跳关系 → 原典证据”。

全流程不调用项目本地 LLM 或 embedding。LLM 只可在 APP 最后组织回答时使用，不能参与原典拆解、证据选择或质量升级。

## 来源与校勘策略

- `docs/wiki/` 原文件永久保留，不静默覆盖；
- 网络来源进入 `docs/wiki_sources/<book>/`，每本书有 `manifest.yaml`、SHA-256、页数、URL、角色与许可；
- 公版古籍扫描本是权威底本；现代排印 PDF 只用于快速定位和 MinerU 初稿；ctext 用户录入本只用于交叉比对；
- 任何未经逐页人工核对的网络 OCR 一律为 `review-needed`；
- 修订决策写入 `corrections.yaml`，保留旧尾文、替代来源、校验锚点和状态。

## 图谱 V2

`graph.json` 升为 schema version 2。原文节点增加完整文本、书名、章节、页码、顺序、提取方式和概念；概念节点增加关键词元数据；方法节点增加输入、步骤、输出、限制；规则节点增加条件、结论、例外、流派和概念。

跨书规则若命中同一概念且结论倾向相反，只生成 `differs_from` 且状态为 `review-needed`。系统不自动判定冲突双方谁正确。

## APP 检索

运行时直接加载 `knowledge_wiki/graph.json`：

1. 从用户问题和当前殿堂主题识别 APP 意图；
2. 以概念名、别名和关键词命中概念；
3. 沿封闭关系词表扩展一跳相关概念；
4. 汇总概念证据和原文关键词得分；
5. 默认只返回 `verified` 原典；
6. 图谱不可用时退回数据库关键词检索，但不调用 embedding；
7. 无可靠命中时不返回任意古籍段落，由对话层使用不带伪造出处的普通劝勉。

引用对象返回 `sourceId`、`quality`、`concepts`、`intent`、`path` 和 `relationHops`。前端引用卡显示质量与概念标签。

## 验收

- 每个网络文件页数和 SHA-256 已记录；
- MinerU 缓存可复用，重新构建不重复 OCR；
- 图谱无悬空边、知识节点均有证据；
- `review-needed` 不进入默认问卦上下文；
- 检索图谱命中时不访问 LLM/embedding；
- API 测试、Ruff、前端 TypeScript 检查通过；
- PDF 至少抽查开头、中间、结尾和修复锚点。
