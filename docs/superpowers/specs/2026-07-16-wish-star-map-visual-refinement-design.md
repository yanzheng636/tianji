# 愿星图视觉修订设计

## 背景与目标

本修订延续现有“个人愿星图”设计，只解决当前首屏的三个视觉问题：背景轨道、轴线和星间连线叠加后过于杂乱；愿望详情与被点击的星星缺少稳定的视线关系；星体尺寸过小，辨认和点击都容易疲劳。

## 已选方案

采用“安静星野 + 悬停阅读面板”方案。

- 移除星图中的同心轨道、纵向轴线和背景轮廓线，但保留愿星之间表达时间关系的连接线。
- 背景只保留低对比度星尘与极弱的局部光晕，用明暗而不是线条建立空间感。
- 愿望详情位于星图右下区域，但默认隐藏；鼠标悬停某颗星时显示对应详情，离开星星后立即隐藏。键盘聚焦星星时采用相同行为。
- 星体视觉直径从 5–9px 提升至 12–18px；进行中愿星使用暖金中心和柔和光晕，已还愿星使用低饱和灰绿。
- 星星的实际点击热区不小于 32px，视觉尺寸与交互尺寸分离，避免为了可点击性把所有星体画得过重。
- 当前悬停或聚焦的星增加清晰但克制的外环，并通过亮度与轻微放大建立层级。

## 布局与交互

详情面板保留在星图内部右下侧，与左侧主标题错开。面板宽度、正文和日期字号保持易读，但只作为星星悬停或键盘聚焦反馈出现。桌面端保持固定出现位置；窄屏时面板移动到星图底部并占据主要宽度。

星体位置继续由现有稳定哈希生成。连接线按愿望时间顺序连接相邻星体，只承担关系表达，不恢复装饰性轨道。悬停预览状态只存在于前端，不改变愿望顺序、创建或还愿流程。

## 视觉约束

- 保留现有墨绿、骨白、铜金和灰绿色体系。
- 背景装饰不得与星体争夺对比度。
- 页面只保留一个主要视觉焦点：当前悬停或聚焦的愿星。
- 动效继续遵守 `prefers-reduced-motion`；悬停与选中不造成明显位移。

## 验证

- 多颗愿星状态下，背景不再出现穿过正文或详情区域的长线。
- 星星在常规桌面分辨率下无需眯眼即可辨认，并能可靠点击。
- 连接线清晰可见，但对比度低于愿星，不与背景争抢注意力。
- 悬停或聚焦不同星星时，用户始终在固定位置看到对应愿望；离开后详情隐藏。
- 进行中、已还愿、当前预览三种状态可仅凭视觉区分。
- 980px、700px 与约 390px 宽度下详情面板不遮挡主要操作，不产生横向溢出。

## 不在本次范围

- 不调整愿望创建、还愿、数据加载或错误处理。
- 不增加星星拖拽、缩放、筛选或独立详情页。
- 不改变下方“尚未还愿”列表。
