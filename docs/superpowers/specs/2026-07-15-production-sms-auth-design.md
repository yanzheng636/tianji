# 生产短信登录链路

## 目标

让手机号登录在生产环境真正调用短信服务；本地仍可用 `console` 联调，但生产不得回传或打印验证码。

## 方案

- 使用阿里云 `SendSms` 官方 RPC/POP 签名协议，通过现有 `httpx` 发起 HTTPS 请求；凭证、签名、模板和 endpoint 全部来自环境变量。
- 生产配置启动时校验 `JWT_SECRET`、`SMS_PROVIDER=aliyun` 和短信凭证；配置不完整直接失败。
- Redis 使用 `SET NX EX` 建立 60 秒冷却闸门；短信发送成功后才写验证码。供应商异常会删除冷却闸门并回滚日计数，返回 503。
- Redis 在生产不可用时不降级到进程内存储，避免多副本登录状态不一致。
- 增加生产环境变量模板、短信失败回滚测试与限流测试。

## 验收

- 发送成功返回 `ok=true`，验证码只存在 Redis，不出现在生产 HTTP 响应。
- 阿里云返回非 `OK`、超时或 HTTP 错误时返回可读的服务不可用错误，并允许立即重试。
- `pytest` 与 Ruff 通过。
